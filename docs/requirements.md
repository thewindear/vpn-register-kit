# VPN Register Kit Requirements

## Goal

Build a lightweight VPN node installer and subscription discovery service. A new node can install and configure `sing-box` plus `nginx`, optionally register itself to a central service, and clients can fetch all registered nodes through a single subscription URL.

## Node Installer

File: `install-node.sh`

- Interactive inputs:
  - `node_id`
  - node name
  - region
  - domain
  - Let's Encrypt email
  - Trojan password, generated by default
  - whether to enable Shadowsocks
  - Shadowsocks port, default `8080`
  - Shadowsocks password, generated by default
  - whether to register to a registry service
  - registry base URL
  - register token
- Install dependencies on remote Ubuntu/Debian servers only:
  - `sing-box`
  - `nginx`
  - `certbot`
  - `ufw`
  - `curl`
  - `jq`
- Configure:
  - nginx public `80/tcp`
  - nginx local fallback `127.0.0.1:8081`
  - sing-box Trojan `443/tcp`
  - optional sing-box Shadowsocks `8080/tcp`
  - Trojan fallback to `127.0.0.1:8081`
  - UFW rules for `22/tcp`, `80/tcp`, `443/tcp`, and optional Shadowsocks port
- Safety:
  - Linux-only guard
  - `--dry-run` mode
  - `--non-interactive` mode with explicit node parameters
  - backup existing configs before replacement
  - sensitive files mode `0600`
  - validate `node_id`, domain, and port range
  - reuse existing protocol passwords from `/root/vpn-sub-kit/node.json` by default
- Verification:
  - DNS check against current public IP
  - `nginx -t`
  - `sing-box check`
  - `systemctl is-active nginx`
  - `systemctl is-active sing-box`
  - listening port check
- Local outputs:
  - `/root/vpn-sub-kit/node.json`
  - `/root/vpn-sub-kit/shadowrocket.txt`
  - `/root/vpn-sub-kit/clash.yaml`
  - `/root/vpn-sub-kit/sing-box.json`
- Optional registration:
  - `POST <registry_url>/register`
  - `Authorization: Bearer <register_token>`
  - JSON payload includes `schema_version: 1`

## Registry Service

Directory: `go-registry/`

HTTP endpoints:

- `GET /health`
- `POST /register`
- `GET /sub?token=xxx`
- `GET /sub?token=xxx&format=shadowrocket`
- `GET /sub?token=xxx&format=base64`
- `GET /sub?token=xxx&format=clash`
- `GET /sub?token=xxx&format=sing-box`
- `GET /api/nodes`

Authentication:

- `/register`: `Authorization: Bearer <register_token>`
- `/sub`: `?token=<subscribe_token>`
- `/api/nodes`: `Authorization: Bearer <register_token>`

Configuration:

- Default config path: `./config.json`
- If config does not exist, create it with:
  - one register token
  - two subscribe tokens
  - `listen: ":8088"`
  - `public_base_url`
  - `data_file: "./nodes.json"`
- Existing config must be reused without regenerating tokens.
- Config file mode should be `0600`.

Node storage:

- Default data path: `./nodes.json`
- Data file mode should be `0600`.
- Nodes are keyed by `node_id`.
- Re-registering the same `node_id` overwrites the previous node.
- Writes must be serialized and atomic.
- Reject oversized request bodies.
- Validate `node_id`, `host`, and port range.

Subscription output:

- Default and `shadowrocket`: line-separated `trojan://` and `ss://` links.
- `base64`: base64-encoded default link output.
- `clash`: Clash/Mihomo YAML with `proxies`, `proxy-groups`, and `rules`.
- `sing-box`: sing-box JSON with basic `dns`, `outbounds`, and `route`.

Response headers:

- `shadowrocket`: `text/plain; charset=utf-8`
- `base64`: `text/plain; charset=utf-8`
- `clash`: `text/yaml; charset=utf-8`
- `sing-box`: `application/json; charset=utf-8`

Logging:

- Log service startup.
- Log generated config path.
- Log `/register`, `/sub`, and `/api/nodes` requests.
- Log token validation failures.
- Log node save failures.
- Never log full tokens or protocol passwords.
- Token logs show only first four and last four characters.

## Registry CLI

Single Go binary commands:

```bash
vpn-registry server -config ./config.json

vpn-registry token list -config ./config.json
vpn-registry token create -config ./config.json
vpn-registry token delete -config ./config.json -token xxx

vpn-registry node list -config ./config.json
vpn-registry node show -config ./config.json -id us-la-001
```

CLI behavior:

- `server`: start HTTP service and generate config if missing.
- `token list`: show register and subscribe token hints only.
- `token create`: add one subscribe token, save config atomically, and print the full new token once.
- `token delete`: delete a subscribe token and save config atomically.
- `node list`: show registered node summaries without passwords.
- `node show`: show one node with passwords masked.

## Client Compatibility

- Shadowrocket: default or `format=shadowrocket`
- v2rayN: default or `format=base64`
- v2rayNG: default or `format=base64`
- ClashX: `format=clash`
- Clash Verge / Verge Rev: `format=clash`
- Mihomo Party: `format=clash`
- Stash: `format=clash`
- sing-box: `format=sing-box`
- NekoBox: default, `base64`, or `sing-box`

## Out Of Scope For Version 1

- Web dashboard
- User groups
- Node groups
- Traffic statistics
- Active health probing
- Node disable/expiry
- Database
- Register token CLI management
- Automatic HTTPS setup for the registry service itself
